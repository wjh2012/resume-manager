# Backlog

현재 로드맵(Phase 0~7)에 포함되지 않은 향후 검토 항목을 기록합니다.

## 면접 기능 확장

- [ ] 면접 결과 평가 (AI가 답변 품질/태도 등을 평가하고 피드백 제공)
- [ ] 예상 면접 질문 생성 (업로드된 문서 기반으로 예상 질문 목록 생성)
- [ ] 면접 컨텍스트에 자기소개서 반영 (에디터에서 작성한 자기소개서를 면접 참고 문서로 선택 가능하게)
- [ ] 면접관 커스터마이징
  - 생성 방식: 자연어 프롬프트로 자유 생성 + UI 컨트롤(게이지바, 선택지)로 구조화 생성
  - 난이도: 게이지바로 조절 (쉬움 ~ 어려움)
  - 면접 유형: 직무면접 / 인성면접 / 임원면접 / 압박면접 등 카테고리 선택
  - 분위기: 친절 / 중립 / 엄격 등 스타일 선택
  - 생성된 면접관 프로필 저장 및 재사용
- [ ] 다중 면접관 (패널 면접)
  - 면접관 2~3명이 턴마다 교대로 질문 (예: 기술면접관 → 인성면접관 → 임원)
  - 하나의 대화에서 진행되므로 면접관 간 이전 대화 맥락 공유
  - UI에서 현재 발언 면접관 표시 (이름/역할 배지)
- [ ] 면접 평가표 및 이전 면접 참조
  - 면접 종료 시 AI가 평가표 생성 (질문별 답변 평가, 종합 피드백, 개선점)
  - 새 면접 생성 시 참조 자료 선택에 이전 면접 평가표 포함 가능
  - 다음 면접관이 이전 평가를 바탕으로 부족한 부분을 집중 질문

## 이력서 기능 확장

- [ ] 커스텀 섹션 (기타사항) — 사용자가 원하는 항목을 자유롭게 추가할 수 있는 섹션 (예: 수상경력, 봉사활동, 어학, 병역 등)
- [ ] 블록 조합 — 여러 이력서에서 섹션을 선택하여 새 이력서로 조합
- [ ] 섹션 순서 커스터마이징 — 학력/경력/기술/프로젝트/자격증 등 섹션의 배치 순서를 사용자가 자유롭게 변경 (예: 경력을 학력보다 위로)

## 채팅 기능 확장

- [ ] 채팅 effort 레벨 — tool use step 수를 사용자가 effort 레벨로 조절 (현재는 `선택 문서 수 + 커리어노트 수 + 2`, 상한 15로 동적 계산)
- [ ] 채팅 2단계 파이프라인 — 1단계: 저렴한 모델로 구조화 분석(generateObject, 읽을 문서 판단, 커리어노트 비교, 대화 압축 필요 여부), 2단계: 분석 결과에 따라 서버가 도구 실행 + 오래된 대화 요약 → 본 응답 생성. 핵심 판단을 LLM 프롬프트가 아닌 JSON 스키마 + 서버 로직으로 처리
- [ ] 면접 코치 에이전트 — 면접관 역할과 분리된 코칭 에이전트. 기존 커리어노트를 참고하여 중복 없이 새 경험을 기록하고, 면접 피드백을 제공
