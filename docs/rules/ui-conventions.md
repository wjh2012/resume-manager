# UI 규칙

## LLM이 자주 놓치는 패턴

AI 코드 생성 시 기능 구현에 집중하느라 UX 맥락을 빠뜨리는 경우가 반복된다. 아래 항목은 사람이라면 당연히 포함하지만 LLM이 습관적으로 누락하는 것들이다.

### 네비게이션 맥락

- **편집/작성/상세 페이지에는 목록으로 돌아가는 버튼을 반드시 포함한다.** 사이드바에 링크가 있더라도, 현재 화면의 맥락에서 뒤로 갈 수 있어야 한다. (예: 상단에 ← 아이콘 버튼)
- 새 페이지를 만들 때 "이 페이지에 어떻게 도착하고, 여기서 어디로 갈 수 있는가?"를 반드시 고려한다.

### 레이아웃 적합성

- 새 페이지를 만들 때 기존 레이아웃 제약(overflow-hidden, 고정 높이 등)을 확인하고 콘텐츠가 잘리지 않는지 검증한다.
- form 요소의 resize가 레이아웃을 깨뜨릴 수 있으면 `resize-none`을 적용한다.

## 컴포넌트 사용 규칙

### 날짜 입력

- 년-월 선택이 필요한 경우 `components/ui/monthpicker.tsx` (MonthPicker)를 Popover와 함께 사용한다.
- 날짜/시간 표시는 `Intl.DateTimeFormat`을 사용하여 브라우저 로케일에 맞게 자동 포맷한다. 하드코딩된 날짜 형식을 사용하지 않는다.
